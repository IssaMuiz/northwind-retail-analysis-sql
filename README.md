# NORTHWIND RETAIL ANALYSIS USING (SQL PROJECT)

## Project Overview

This project is a hands-on SQL data analysis case study using the Northwind retail database.
The goal is to simulate a real-world business environment where we analyze customer behavior, sales performance, and product revenue using SQL.

The project focuses on building strong fundamentals in:

* SQL querying
* Database relationships (JOINs)
* Aggregations and KPIs
* Business insights generation

## Business Questions to be Answered

This project answers key business questions such as:

* Who are the most valuable customers?
* Which products generate the most revenue?
* What are the top-selling products?
* How do customers contribute to total revenue?
* How are tables related in a relational database?


## Database Structure

The analysis is built on the following core tables:

* `customers`
* `orders`
* `order_details`
* `products`

### Key Relationships

* `customers.id → orders.customer_id`
* `orders.id → order_details.order_id`
* `order_details.product_id → products.id`

These relationships form the backbone of the analysis pipeline.

## Key Concepts Practiced

During this project, the following SQL concepts were applied:

* SELECT statements
* JOIN operations (INNER JOIN)
* GROUP BY aggregation
* SUM() for revenue calculation
* ORDER BY for ranking results
* Data aggregation across multiple tables


## Core Analysis Performed

### 1. Customer Revenue Analysis

Calculated total revenue generated by each customer:

* Joined `customers → orders → order_details`
* Computed revenue using:

  ```
  revenue = quantity × unit_price
  ```
* Aggregated results per customer
